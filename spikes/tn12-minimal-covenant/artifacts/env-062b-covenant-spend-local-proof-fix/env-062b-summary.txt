ENV-062B covenant-spend local proof fix

Result: FIXED_OFFLINE

Root cause:
The covenant redeem script encoded the UPSTREAM VERSION-0 previous-txid proof
(TransactionID(prev_rest || prev_payload) via keyed-blake2b), copied from the
official tn10-toc3 crypto/txscript/examples/covenants.rs counter example. The
live ENV-060C create transaction is TX_VERSION_TOCCATA / version 1, whose txid
is composed as TransactionV1Id(payload_digest || rest_digest) where
  rest_digest    = TransactionRest(transaction_v1_rest_preimage(tx))
  payload_digest = PayloadDigest(tx.payload)
and where TransactionRest / PayloadDigest / TransactionV1Id are KEYED-BLAKE3
hashers (domain string zero-padded to 32 bytes), not keyed-blake2b. The v0 script
therefore computed the wrong previous-txid and failed at the first OP_EQUALVERIFY
with TxScriptError::VerifyError. See root-cause.txt for the full classification.
Primary classification: wrong redeem script / wrong tx-id preimage convention
(v0 used for a v1 tx) plus the keyed-blake2b-vs-keyed-blake3 domain detail.

Scope note (important):
The existing live ENV-060C UTXO is locked by the v0-style script and is
structurally unspendable by that script under a v1 create tx. Per the ENV-062A
recommendation, ENV-062B does NOT try to retroactively spend that UTXO. The fix
is a v1-compatible covenant script/state-transition design, proven to pass the
covenant VM offline. The v0 build_covenant_script and the ENV-060C reconstruction
are left byte-for-byte unchanged so the live create txid still reproduces.
