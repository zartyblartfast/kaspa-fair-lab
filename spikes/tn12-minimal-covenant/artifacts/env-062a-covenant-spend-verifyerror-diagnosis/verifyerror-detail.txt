Root cause:
The live ENV-060C create transaction is TX_VERSION_TOCCATA / version 1. The official upstream covenant example is a version-0 in-VM example: it uses transaction_v0_id_preimage(tx), splits that preimage into rest + payload, and its covenant script recomputes TransactionID(prev_tx_rest || prev_tx_payload) directly.

For the live version-1 create tx, the helper fed transaction_v1_rest_preimage(create_tx) plus create_tx.payload. That is not a version-1 transaction-id preimage. Version-1 txid is TransactionV1Id(payload_digest || rest_digest), where rest_digest is TransactionRest(transaction_v1_rest_preimage) and payload_digest is PayloadDigest(payload). The local debug confirms:
- V1 rest digest: 09ecf22a8928e383efe978b01af61d7f146c6eca5d00782202a2a1796ecd7a4c
- V1 payload digest: 9c0ca2acb45e92ffe6ceb4ae29188b35c82d9676cdd3ce067fd6ccc30a9c4a38
- reconstructed V1 id from payload_digest || rest_digest matches create txid: true

Therefore the helper cannot make the existing live covenant UTXO pass this script check by changing only sigscript stack items, unless it can provide a preimage that hashes under TransactionID to the already-fixed v1 txid, which is not a constructive spend path.
